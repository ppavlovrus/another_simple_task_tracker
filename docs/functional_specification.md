# Функциональные требования к приложению по управлению задачами
## Основные определения
* Пользователь - пользователь системы? который работает с задачами
* **Задача** - сущность в системе, представляющая задачу пользователя.
У задачи есть наименование, описание, статус задачи, срок на выполнение, дата начала выполнения задачи, фактическая дата завершения, объем потраченных на нее временных ресурсов. 
Так же к задаче могут быть приложены дополнительные материалы в виде файлов - документы, аудио, видео, картинки.
* **Статус задачи** - метка, присущая задаче, которая описывает, в каком состоянии задача. Доступны следующие статусы:
  * **Создана** - задача создана, но не взята в работу
  * **Выполняется** - задача выполняется
  * **Приостановлена** - задача приостановлена
  * **Завершена** - задача завершена
  * **Отменена** - задача отменена
* **Жизненный цикл задачи** - последовательность статусов, которые задача должна пройти в ходе своего существования в системе.
Задача не может перескакивать через этапы жизненного цикла
* **Проект** - логически объединенная совокупность взаимоувязанных задач, ограниченная по времени общего выполнения
* **Архив** - хранилище выполненных или отмененных задач
* **Бэклог** - хранилище не начатых задач
* **Доска задач** - экранная форма, на которой отображаются все задачи для пользователя или пользователей, сгруппированные по статусам.
* **Комментарии к задаче** - в целом отдельные сущности, заметки пользователей к задаче
